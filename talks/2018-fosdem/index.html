Event bubble

Show delegated event at the "correct" node

It shows regular event listener on nodes, as well as delegated ones for some libraries like React or jQuery.

Sources

An actual source tree, showing sourcemapped files

You can expand all the children of a node by doing Alt + Click on it. We also have the same shortcut in the inspector.

Outline view

Functions and ES6 classes' methods

So here we can jump to onComponentChange

Source-map

JSX, typescript, C, … highighting

My code is nicely highlited, even jsx parts. It also works on typescript, coffescript, and even C files, which you might be able to see when doing WASM debugging.

Source-map

Link to generated location

Either for the whole file, or on a specific function. Here you can see how JSx is interpreted.

Collapsed frames

Grouped frames by library

Here we are paused and we can see the stacktrace. But instead of having all the React internals calls, they are nicely grouped, and, if I want to, I can expand it. This way, I can focus only on the parts that matters.

Blackbox sources

Stepping will not go into those sources. Useful for libraries / framework

Vars mapping

Evaluate original names, even on ugglified files.

Even if I'm in an original file, and the event inspector-rule-autocomplete-css-variable has been renamed to e, I can still evaluate event and access properties on it.

Continue to here

Instead of stepping multiple times to get where I want, or adding another single-use breakpoint, I can simply right click on the line I want to go to, and select "continue to here".
