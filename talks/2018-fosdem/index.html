Continue to here

So, usually, when you debug something, it involves a lot of clicks on step over, step-in, step-out function so you can get to the actual faulty line. I sometime even do add a breakpoint 5 lines after the paused line so I don't haveto click multiple times. Now in the debugger it's even simpler because I can right click on the line I want to go to, and select "continue to here". In my case, I did not even had to do that, because it was the first line that was faulty with "this" being undefined. And I know why it is undefined, it's because I did not bound this to my method, like React ask you too.

AST breakpoints

Let the debugger handle your breakpoints

So, here I added a constructor to bind my method, and I reloaded my page. Do you see what happened ? Even if I added 4 lines, the debugger is still where I want it to be, on the "searchBugs" variable assignment. It's because the debugger has what we call AST breakpoints. Which means, breakpoints are not only referenced by a file and a line number, but also by an ast location. Here it is the "searchBug" variable assignment in the onComponentChange method. I find this very useful because you can tweak your code and not have to deal with your breakpoints being on a whole other expression you don't care about.

Styled group

Used by Redux logger

Since I added redux-logger to my app, I can see my actions being dispatched in a nice way thanks to the conjunction of 2 things we added in the new console.

console
.group


console.group("A");
console.log("My log");
console.groupCollapsed("B");
console.log("Another log");
console.groupEnd();
console.groupEnd()
console.group was already supported in the old console, but you could't collapse a group, which was making them almost useless. Now you can expand and collapse them as you wish. This allowed us to fully support console.groupCollapsed, which start a group in a collapsed state.

Styled
message


console.log(
  "%cHello %cWorld",
  "color: red",
  "color: blue"
)
The second thing we added was styled console.group. We already have styled console.log, but not styled group. This is now fixed. If you don't know styled console message, it's a nice little thing that let you style parts of your message using a whitelist of CSS properties. The way you do this is by declaring a style unit, with %c, and css properties on the last arguments of console.log calls. This way you can build fun helper function to have rainbow logs.

Network logs

Status code MDN link. Shared components with the Netmonitor

One last thing I wanted to show you in the console is that we can display network logs. We share the same components with the netmonitors so you should have a consistent experience using those 2 tools. One thing is different though, you can click on the status code in the console to get to the MDN page of said status code, in case you wonder what the 200 code means.

NETWORK

So I saw the xhr call to Github, let's switch to the netmonitor to fully inspect it.

Photon colors

Rewritten in React & Redux. Disable cache + Persist log

The netmonitor was also rewritten in React and Redux and got these nice Photon colors as well. There were some improvements in the UI, so you can disable the cache and persist log right into the panel.

Headers

Links to MDN for status code and Headers

When clicking on a network entry, you get what we call the detail panel which let you inspect it more deeply. Here you can see all the request and response headers, with links to the approriate MDN pages when they exists.

Stacktrace

Sourcemapped, open the debugger when clicked

You can also see a soucemapped stacktrace to know where this network call was made from.

Pause

Pause / Resume network recording

And if you already found what you were looking for and are disturbed by new logs coming in, you can simply pause the network recording. Network calls will still be made by the browser, but they won't show up in the netmonitor.

So that's it. Well aslmost. I want to highlight a few things that wer not made by the DevTools team but that can help developers doing their jobs.

WebExtensions support

React, Redux, Angular, …
First is DevTools webextensions support. the extension team, with sometime some help from us, did a tremedeous job so we can benefit from all the chrome extensions that are already out there. So things like React Devtools, Redux Devtools, Angular works directly in Firefox. here is a screenshot with the website I almost successfully switched to React.

Firefox Screenshot

Second is Firefox Screenshot. This you don't need to install, it is baked-in Firefox. It tries to find the container when you mouse over something, and you can then save locally, to your clipboard or to the cloud where you can assign it a life span. My favorite workflow is copy to clipboard then paste into Slack or Sketch. No more temp files everywhere in your Desktop.

Multi-Account Containers

And the last shout-out is for the Multi Account containers extension. It let you have multiple "identitites" in the same browser. It is very useful if you are working on a site with different user roles. So here you can set them as container, and not have to log out and log in again with a user with the correct role. Super helpful.
